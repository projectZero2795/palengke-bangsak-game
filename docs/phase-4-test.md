# Phase 4 Test Notes

## Scope

Phase 4 makes the placeholder player visibly animate while moving.

Included:

- `PlayerAnimationController`
  - swaps between idle sprites and walk sprites;
  - supports 8-direction facing:
    - down;
    - down-right;
    - right;
    - up-right;
    - up;
    - up-left;
    - left;
    - down-left;
  - uses configurable `framesPerSecond`;
  - tracks `PlayerFacingDirection`;
  - selects a direction-specific sprite instead of sliding a front-facing sprite;
  - keeps the last facing direction when idle.
- `PlayerFacingDirection`
  - `Down`;
  - `DownRight`;
  - `Right`;
  - `UpRight`;
  - `Up`;
  - `UpLeft`;
  - `Left`;
  - `DownLeft`.
- Directional placeholder sprites:
  - generated by `tools/generate_phase4_directional_player_sprites.py`;
  - 8 directions;
  - idle + 4 walk frames per direction;
  - red, green, blue, and yellow variants.
- Default player prefab updates:
  - `PlayerAnimationController`;
  - red fallback idle/walk references;
  - red directional idle references;
  - red directional walk references;
  - `walkFramesPerDirection = 4`.
- EditMode tests for:
  - horizontal direction resolution;
  - vertical direction resolution;
  - diagonal direction resolution;
  - stationary fallback direction;
  - no left/right fallback flip when directional sprites exist;
  - walk frame looping;
  - directional walk sprite indexing;
  - walking-vs-idle threshold;
  - prefab animation wiring;
  - `PrototypeMap` playable-player animation wiring.

Not included:

- final production art;
- directional north/south sprite sheets;
- Unity Animator graph;
- Bang action;
- Tag action;
- caught state;
- role system;
- round rules;
- Photon;
- WebGL build;
- Docker/Kubernetes/Argo files.

## Why this uses a script-based animation controller

The current sprites are generated placeholders. A script-based controller is easier to review now because it proves the animation rules:

- idle when not moving;
- walk frame cycling when moving;
- facing state changes from input direction;
- direction-specific sprite selection.

A Unity Animator graph can still be introduced later if the final sprite sheet needs more complex animation states.

## Manual Unity Editor checks

1. Pull latest `main`.
2. Fully close and reopen Unity so project settings and prefab changes reload.
3. Open `unity/` in Unity 2022.3.50f1 or newer.
4. Open `Assets/Scenes/PrototypeMap.unity`.
5. Press Play.
6. Move with keyboard:
   - `W` / up arrow;
   - `A` / left arrow;
   - `S` / down arrow;
   - `D` / right arrow.
7. Move diagonally using key combinations:
   - `W + D`;
   - `W + A`;
   - `S + D`;
   - `S + A`.
8. Confirm the player faces the movement direction, including diagonals.
9. Confirm the player cycles through walk frames while moving.
10. Confirm the player returns to the matching idle direction when stopped.
11. Drag the lower-left joystick placeholder and confirm it also drives directional animation.
12. Confirm Console has no red errors.
