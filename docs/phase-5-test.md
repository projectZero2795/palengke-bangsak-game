# Phase 5 Test Notes

## Scope

Phase 5 designs the safe local `Bang` action.

Included:

- `BangActionVisualStyle`
  - `FingerMarker`;
  - `ToyDart`;
  - `CartoonLightBeam`;
  - `TsinelasMarker`.
- `BangActionController`
  - safe cartoon Bang marker;
  - range indicator;
  - cooldown;
  - keyboard trigger with `Space`;
  - effect direction based on the current player facing direction;
  - no damage, no catching, no hit validation.
- `BangActionHud`
  - runtime-created `BANG!` button;
  - compact lower-right screen anchoring;
  - cooldown text while the action is unavailable.
- Placeholder assets:
  - `bang_marker_placeholder.png`;
  - `bang_range_placeholder.png`;
  - generated by `tools/generate_phase5_bang_assets.py`.
- Default player prefab updates:
  - `BangActionController`;
  - `BangActionHud`;
  - cartoon marker sprite;
  - range indicator sprite.
- EditMode tests for:
  - safe default visual style;
  - cooldown behavior;
  - cooldown progress;
  - 8-direction effect vectors;
  - effect placement;
  - prefab wiring;
  - `PrototypeMap` playable-player wiring.

Not included:

- projectile physics;
- raycast hit detection;
- player hitboxes;
- caught state;
- Taya/Hider role restrictions;
- score;
- networking;
- Photon;
- WebGL build;
- Docker/Kubernetes/Argo files.

## Safety note

This phase does not implement realistic weapons. `Bang` is represented as a playful cartoon marker/light-burst effect. The visual-style enum keeps future safe alternatives explicit, including toy dart and tsinelas marker themes.

## Manual Unity Editor checks

1. Pull latest `main`.
2. Fully close and reopen Unity so project settings and prefab changes reload.
3. Open `unity/` in Unity 2022.3.50f1 or newer.
4. Open `Assets/Scenes/PrototypeMap.unity`.
5. Press Play.
6. Confirm a compact `BANG!` button appears fully inside the lower-right of the screen, without clipping off the edge.
7. Confirm a soft range indicator appears around the player.
8. Move in any direction and press `Space`.
9. Confirm the cartoon Bang marker appears in the direction the player is facing.
10. Press the `BANG!` button.
11. Confirm the cooldown prevents spam and the button displays cooldown text.
12. Confirm the action does not catch, damage, delete, or affect another player.
13. Confirm Console has no red errors.
