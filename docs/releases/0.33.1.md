# Bang-Sak 0.33.1 Photon room connectivity hotfix

Released: 2026-07-12

## Problem

Some WebGL clients repeatedly showed `Photon connection failed: Error` while
creating room `1234`. The affected resolver path could not reliably follow
Photon's global name-server alias even though the direct EU Photon endpoint and
the deployed App ID were healthy.

## Fix

- WebGL connects through the direct EU secure Photon name-server route that
  matches the game's fixed `eu` region.
- Room create/join makes up to two attempts instead of failing after the first
  transient route error.
- Connection callbacks no longer destroy the runner while `StartGame` is still
  returning its failure result and preparing the retry.
- The room UI reports the last failure and tells the player when it is retrying.
- Privacy-safe browser logs identify the direct EU route without exposing the
  Photon App ID or credentials.
